# 实施 Audience Manager {#implementing-audience-manager}

本节概述并说明与Audience Manager数据管理平台([!DNL DMP])快速入门相关的流程。 此部分旨在帮助业务团队、项目经理和技术经理了解Audience Manager实施过程。 Audience Manager入门大约需要六周到三个月的时间，具体取决于您的数据收集需求。

我们的实施技术有助于与新客户建立协商式合作关系。 此过程旨在：

* 了解并了解您的业务需求；
* 制定可行计划，以满足这些要求；
* 开发自定义解决方案，以帮助满足独特要求或用例；
* 确保您的专有数据已导入并在Audience Manager中提供。

我们的合作伙伴解决方案和客户管理团队在实施过程之前、期间和之后将与您密切合作。

Audience Manager采用分阶段的方法来设置和实施。

## 构建、测试和培训阶段{#build-test-train-phase}

在构建、测试和培训阶段，您将与指定的合作伙伴解决方案主管一起审查数据收集策略和原型。

<!-- c_build_test_train.xml -->

您的数据收集策略将进行端到端的[!DNL QA]测试。 合作伙伴解决方案将跟踪发现的错误并与我们的系统工程师协调问题解决方案。 客户培训可以与这些其他工作同时开始。

下表介绍了在此阶段进行的关键活动：

| 活动 | 用途/描述 | 参与者 |
|---|---|---|
| 准备数据收集策略 | 与Adobe技术团队合作，制定符合您业务要求的数据收集计划 | 业务和技术团队 |
| 部署和测试代码 | 在暂存环境中测试建议的解决方案并执行跨浏览器测试 | 技术团队 |
| 验证功能并解决错误 | 检查和传达结果、解决错误并重新测试 | 技术团队 |
| 用户培训 | 提供有关Audience Manager功能、工具和报告的教育和了解 | 业务团队 |

此阶段的交付内容可包括：

* 已完成和已接受的数据收集计划；
* 端到端[!DNL QA]测试；
* Audience Manager用户界面功能的基本说明；
* 接受和签收。

## 代码实现{#code-implementation}

Audience Manager使用Data Integration Library([!DNL DIL])接收受众数据。

有关如何使用[!DNL DIL]的详细说明，请参阅[获取和实施DIL代码](../dil/dil-overview.md)。

### 参与者

合作伙伴解决方案可以直接与您的技术团队合作，以帮助部署代码、解决最终问题并满足其他要求。

## 实施后支持{#post-implementation-support}

我们的协作努力不会随着最终部署而停止。 实施完成后，我们的客户管理团队将接手。

产品实施过程完成后，客户经理将继续提供支持和咨询服务。 您可以期望与客户经理定期会面。 这些会议可确保您从Audience Manager中获得最大的使用量和价值。

请在[此处](https://www.adobe.com/products/audiencemanager.html)联系我们，以了解更多信息并开始使用Audience Manager。
