# Implementing Audience Manager {#implementing-audience-manager}

This section outlines and explains the processes related to getting started with the Audience Manager data management platform ([!DNL DMP]). 本节旨在帮助业务团队、项目经理和技术经理了解Audience Manager实施流程。根据您的数据收集需求，Audience Manager快速入门大约需要六周到三个月。

我们的实施技术有助于与新客户建立咨询合作关系。此过程旨在：

* 发现和了解您的业务需求；
* 制定可满足这些需求的切实可行的计划；
* 开发自定义解决方案以帮助满足独特的要求或使用案例；
* 确保您的专有数据已导入并在Audience Manager中提供。

我们的合作伙伴解决方案和帐户管理小组将在实施过程之前、之中和之后与您紧密合作。

Audience Manager采用分阶段的设置和实施方法。

## Build, Test, and Train Phase {#build-test-train-phase}

在构建、测试和培训阶段，您将通过指定的合作伙伴解决方案潜在客户审阅数据收集战略和原型。

<!-- c_build_test_train.xml -->

Your data collection strategy will undergo end-to-end [!DNL QA] testing. 合作伙伴解决方案将跟踪发现的错误并与我们的系统工程师协调问题解决。客户培训可以与这些其他工作并驾齐驱。

下表介绍了在此阶段进行的关键活动：

| 活动 | 用途/说明 | 参加者 |
|---|---|---|
| 准备数据收集战略 | 与Adobe技术团队合作构建满足业务需求的数据收集计划 | 业务和技术团队 |
| 部署和测试代码 | 在分阶段环境中测试提议的解决方案并执行跨浏览器测试 | 技术团队 |
| 验证功能并解决缺陷 | 检查和交流结果、解决缺陷和重新测试 | 技术团队 |
| 用户培训 | 为Audience Manager功能、工具和报表提供教育和理解 | 业务团队 |

此阶段的交付内容包括：

* 完整且接受的数据收集计划；
* End-to-end [!DNL QA] testing;
* Audience Manager用户界面功能的基本说明；
* 接受和签字。

## Code Implementation {#code-implementation}

Though the deployment process may seem complex, the code implementation is as simple as adding a few lines of [!DNL JavaScript] adjacent to the closing `</body>` tag of your website.

<!-- c_code.xml -->

### 部署

The Audience Manager code snippet calls [!DNL Akamai] to download the business rules set up previously in the user interface. 此外，客户端浏览器可缓存此信息，从而帮助减少页面和服务器负载时间。我们的代码和数据收集方法旨在维护跨库存的用户体验。

### 参加者

合作伙伴解决方案可以与您的技术团队直接合作，帮助部署代码、解决最终问题并满足其他要求。
