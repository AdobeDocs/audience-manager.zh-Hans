# Documentation Updates 2019 {#documentation-updates}

2019年发布的一列表新文件或修订文件。 包括对受众管理器文档的次要更新，这些更新可能在Experience Cloud发行说明中未涵盖。

有关对 [其他解决方案的更改](https://docs.adobe.com/content/help/en/release-notes/experience-cloud/current.html) ，请参 [!DNL Audience Manager] 阅Experience Cloud发 [!DNL Experience Cloud] 行说明。 有关旧版 [本说明](../docs-updates/docs-2018.md) ，请参阅“上一年”部分。

## November, 2019 {#aam-november-updates}

| 主题 | 描述 |
|--- |----|
| [加利福尼亚消费者隐私法(CCPA)支持和隐私文档大修](../overview/data-security-and-privacy/data-privacy.md) | 2020 [年1月1日生效的《加州消费者隐私法》](https://www.caprivacy.org/about)(California Consumer Privacy Act, CCPA)将为加利福尼亚州居民提供有关其个人信息的新权利，并将对在加利福尼亚开展业务的某些实体承担数据保护责任。 <br> <br> 受众管理器通过Adobe Experience Platform Privacy Service等隐私工具帮助您遵守隐私法规 [中规定的义务](https://www.adobe.io/apis/experienceplatform/home/services/privacy-service.html) ，以便访问和删除数据。 <br><br> 我们已更新了当 [前的退出管理](../overview/data-security-and-privacy/data-privacy-requests.md#opt-out-requests) 流程，以包括选择退出任何声明的ID（例如CRM ID）。 如果通过声明的ID退出，则声明的ID和最后一个链接的设备将选择退出受众管理器数据收集。 选择退出请求现在还会以批处理和实时方式 [向支持](../overview/data-security-and-privacy/assets/AAM-Partners-October2019.xlsx) 此功能的目标合作伙伴发送不细分请求。 <br><br> 此外，我们重新设计了 [数据安全](../overview/data-security-and-privacy/data-security.md)、 [数据隐私和](../overview/data-security-and-privacy/data-privacy.md)数据治理文档 [](../overview/data-security-and-privacy/data-governance.md) ，使您能更轻松地找到符合上述法规要求的信息。 |

## March, 2019 {#aam-march-updates}

| 主题 | 描述 |
|--- |----|
| [全局数据源](../features/global-data-sources.md) | 所有的 Audience Manager 客户都可以访问全局数据源，这些数据源包含由多家设备制造商（例如，Apple、Samsung、Microsoft、Roku，以及 Android 设备制造商）生成的设备广告 ID。这些 ID 是设备制造商出于广告宣传的目的而提供的。您可以使用全局数据源来同步设备 ID，并导入或导出这些映射中断开映射的数据。Audience Manager 将根据其格式，验证由客户导入的设备广告 ID (DAID)，以确保这些 ID 符合设备制造商列出的标准格式。 |
| [ID 映射限制](../features/administration/usage-limits.md#id-mapping-limits) | 通过限制与设备 ID 关联的跨设备 ID 映射数量，我们改进了分段、报告和导出区段的过程。新近推行的限制有助于我们缓解错误集成和不准确的数据集可能对引入和下游处理造成的影响。<br/>新的限制包括： <ul><li>每个跨设备 ID 最多可关联 100 个设备 ID</li><li>每个设备 ID 最多可关联 10 个跨设备 ID</li><li>1 个设备 ID 最多可关联 1000 个设备 ID</li></ul> |
