# 2019 年文档更新 {#documentation-updates}

2019 年发布了一系列新文档或修订文档。其中包括对 Audience Manager 文档进行的次要更新，这些更新可能未包含在 Experience Cloud 发行说明中。

欲了解对 [!DNL Audience Manager] 和其他 [!DNL Experience Cloud] 解决方案所做的更改，请参阅 [Experience Cloud 发行说明](https://experienceleague.adobe.com/docs/release-notes/experience-cloud/current.html)。对于较早的发行说明，请参阅[往年发行说明](../docs-updates/docs-2018.md)。

## 2019年11月 {#aam-november-updates}

| 主题 | 描述 |
|--- |----|
| [《加州消费者隐私法案》(CCPA) 支持和隐私文档全面修改](../overview/data-security-and-privacy/data-privacy.md) | 2020年1月1日生效的[《加州消费者隐私法案》(CCPA)](https://www.caprivacy.org/about)将为加利福尼亚州居民提供有关其个人信息的新权利，并将对在加利福尼亚州开展业务的某些实体承担数据保护责任。<br> <br> Audience Manager通过诸如Adobe Experience Platform隐私服务之类的用于数据访问和删除请求的隐私工具， [帮助您](https://www.adobe.io/apis/experienceplatform/home/services/privacy-service.html) 遵守隐私法规规定的义务。<br><br> 我们更新了当前 [的选择退](../overview/data-security-and-privacy/data-privacy-requests.md#opt-out-requests) 出管理流程，以包含选择退出任何声明的ID（例如CRM ID）的选项。如果由声明的ID选择退出，则声明的ID和最后一个链接的设备将退出Audience Manager数据收集。 现在，选择退出请求还会实时分批向支持此功能的[目标合作伙伴](../overview/data-security-and-privacy/assets/AAM-Partners-October2019.xlsx)发送取消分段请求。 <br><br> 此外，我们还重新设计了 [数据安全](../overview/data-security-and-privacy/data-security.md)、 [数据隐私](../overview/data-security-and-privacy/data-privacy.md)和 [数据](../overview/data-security-and-privacy/data-governance.md) 管理文档，以便您更轻松地查找遵守上述法规要求所需的信息。 |

## 2019年3月 {#aam-march-updates}

| 主题 | 描述 |
|--- |----|
| [全局数据源](../features/global-data-sources.md) | 所有的 Audience Manager 客户都可以访问全局数据源，这些数据源包含由多家设备制造商（例如，Apple、Samsung、Microsoft、Roku，以及 Android 设备制造商）生成的设备广告 ID。这些 ID 是设备制造商出于广告宣传的目的而提供的。您可以使用全局数据源来同步设备 ID，并导入或导出这些映射中断开映射的数据。Audience Manager 将根据其格式，验证由客户导入的设备广告 ID (DAID)，以确保这些 ID 符合设备制造商列出的标准格式。 |
| [ID 映射限制](../features/administration/usage-limits.md#id-mapping-limits) | 通过限制与设备 ID 关联的跨设备 ID 映射数量，我们改进了分段、报告和导出区段的过程。新近推行的限制有助于我们缓解错误集成和不准确的数据集可能对引入和下游处理造成的影响。<br/>新的限制包括： <ul><li>每个跨设备 ID 最多可关联 100 个设备 ID</li><li>每个设备 ID 最多可关联 10 个跨设备 ID</li><li>1 个设备 ID 最多可关联 1000 个设备 ID</li></ul> |
